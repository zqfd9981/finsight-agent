# 检索策略分类器训练集标注审计（修正版 v2）

> 生成：2026-07-18（深夜） | 数据源：`labeled.jsonl`（301 条）| 对照：用户最终确认的标注口径

> **本版推翻 v1 的关键结论**：v1 把「集运板块逻辑/航运板块估值/创新药出海板块」等标 event 的样本判为“矛盾应翻 dual”，**那是错的**——按用户规则，这些“板块+估值/机制/情绪”本就该 event。本版按修正口径重标。

## 0. 用户最终确认的标注口径（权威）
- **dual_primary** = 回答需要**公司级财务/披露数据**，判定条件（满足任一）：
  1. query 提到**具体公司名**（隆基绿能、宁德时代…）→ dual；
  2. query 为**板块龙头 / 板块业绩 / 板块订单**（涉及多家公司财务数据）→ dual。
- **event_primary** = 纯**板块 / 宏观 / 估值 / 逻辑 / 情绪 / 机制 / 风险偏好**层面，不需要拉具体公司年报 → event。
- **disclosure_primary** = 公司**内生**事件（扩产/业绩预告/分红/订单/产能…），主语落单家公司自身披露。
- **判定轴是“要不要查公司财务”，不是“有没有『板块』这个词”**。

## 2. 对 v1 审计的自我纠错
v1 §3.2/§4 列的「8 条应翻 dual」（rsc_gen_008/195/197/228/219/221/235/236）经用户确认后**全部应维持 event**，理由：它们都是「板块+估值/逻辑/情绪/机制」，按口径本就 event，不存在矛盾。真正的问题是 v1 误用了“板块=dual”的错误前提。

## 4. 用户 query 的真根因（重要）
「红海局势会对**隆基绿能**产生什么影响」= 事件 + **具体单一 A 股公司** → 按口径**本应 dual_primary**（需查隆基年报/披露）。
- 训练集**没有**“红海×隆基绿能”这个跨主题组合（OOD）：红海只与航运公司共现、隆基只与光伏事件共现。
- 但训练**有**「光伏反内卷+隆基=dual」（rsc_ha_015）、「硅料限产+隆基/通威=dual」（rsc_gen_200），证明“隆基+事件=dual”本可学。
- 分类器返回 event_primary（margin=0.307，top2=disclosure）是**把“具体公司”误判成了“板块/纯事件”**：它只吃裸文本，没有结构化的“这是公司名 vs 这是板块名”判别力，看到“红海”就往航运事件靠。
- **所以这不是标签矛盾，是 OOD 组合 + 分类器无实体判别力**。修标签（上面 3.1/3.2 那几条）能小幅改善，但根治要靠见“任意事件×任意公司”组合，或代码层实体兜底。

## 5. 修复建议（按性价比）
- **P2 实体感知兜底（首选，几行代码）**：router 已抽 company 实体 + 意图 event_impact → 强制 dual_primary。直接把用户规则编码进路由，绕开分类器模糊判断。用户 query 立刻生效。
- **P0 KMP 守卫**：`trained_strategy_classifier.py` 加 `KMP_DUPLICATE_LIB_OK=TRUE`（照抄 bge_m3.py），防生产加载撞 OpenMP 静默回退 Stub（同 bge-m3 陷阱）。
- **P1 标签微修 + 补 OOD 样本**：按 §3 重标 4~6 条矛盾样本；补“红海/关税/地缘事件 × 非航运/消费电子公司”等跨主题组合样本，教模型“任意事件+任意公司=dual”。
- **兜底健壮性**：`_should_use_local_rag` 在“外部返 0 条相关”时也触发本地 RAG/披露源，任何未来误判不再整段降级。
