# 检索策略分类器标注规则

> 适用于 `backend/training/retrieval_strategy_classifier/data/labeled/` 下训练数据的标注工作。
> 三分类器对 query 输出 `event_primary` / `disclosure_primary` / `dual_primary` 三种 label。
> 下游两个检索源：**event 源**（Bocha 新闻聚合，返回网页新闻，`company_codes` 字段空）+ **disclosure 源**（CNInfo / SSE 实时全文检索，返回真实公司公告 + 真实股票代码）。

---

## 1. 标签定义

### event_primary
query 答案可以纯靠 event 源（外部新闻 / 政策 / 地缘）+ 宏观行业推论给出，**不需要** A 股具体公司/板块的披露数据。

### disclosure_primary
query 主语锁定单一公司（或少数公司），答案必须依赖该公司的具体披露（公告 / 财报 / 订单 / 扩产 / 分红 / 股权激励等公司层信息）。**event 源几乎用不上**（除非少量背景补充）。

### dual_primary
query 答案**必须**落到 A 股具体公司/板块层数据（估值分位、季报披露、订单数据、公告细节、机构持仓变动等），单靠 event 源 + 宏观推论**明显不够**——必须叠加 disclosure 源的真实公司公告数据。

---

## 2. 核心判定规则（定稿）

只问一个问题：**"这个 query 的答案，如果只跑 event 源（不含任何 A 股具体公司披露数据），能不能给出来？"**

- 能 → **event_primary**
- 不能 → **disclosure_primary** 或 **dual_primary**（再按下面规则细分）

### 三选一细分

| 规则 | label |
|---|---|
| 主语锁定单一（或少数）公司，问的是该公司的具体披露事项 | `disclosure_primary` |
| 答案需要 A 股具体公司/板块层数据（估值分位、季报披露、订单数据、公告细节、机构持仓等），且不锁单一公司（即问"哪些公司" / "板块画像" / "A 股龙头" 等）| `dual_primary` |
| 答案可以纯靠宏观推论给出，不需查 A 股具体公司 | `event_primary` |

**关键判据**：判定不是看 query "提到什么词"，而是看 **query 答案链路是否需要 A 股具体公司层数据**。

---

## 3. 不要做的事

- ❌ 不要用"提到板块 → dual" / "提到公司 → dual" 这种单一维度简化规则（早期 spec 内部矛盾导致的假象）
- ❌ 不要用"query 表面是抽象概念（情绪 / 逻辑 / 估值修复）就归 event"简化判别（被 rsc_gen_249 等样本修正过）
- ❌ 不允许 `unknown` / `other` 标签
- ❌ 不要写"占位符 query"（必须自然中文）
- ❌ 不允许"X 公司 Q3 业绩怎么样"出现 18 次这种笛卡尔积

---

## 4. 边界 case 处理

### 真边界（需要双源 / 哪个源都明显不够 dual）

| query 形态 | 判定 |
|---|---|
| "哪些 A 股股 / 哪些 A 股板块 / 哪些龙头 / 哪些公司" | dual（答案必须有具体公司清单）|
| 提到具体公司名（招商轮船、宁德时代、立讯精密 等）| dual 或 disclosure（按主语锁不锁定单一公司）|
| 提到"A 股板块 / A 股龙头" + 答案需要落到公司画像 | dual |

### 倾向 event（纯宏观推论够用）

| query 形态 | 判定 |
|---|---|
| "哪些行业 / 哪些方向 / 哪些宏观领域"（不锁 A 股）| event |
| 主语是政策意图本身、"政策方向实质变化" 类问法 | event |
| 主语不在 A 股（如"对跨国公司资金运作影响" / "对国内政策窗口期" / "对中国宏观经济"）| event |
| 提问对象是宏观信号（央行利率、汇率、油价、监管框架）| event |

### "抽象市场概念"陷阱

早期误以为"`情绪/逻辑/估值修复/估值锚/博弈属性`等抽象市场概念 → event"——错的。

**真实规则**：即使是抽象概念问法（如"估值修复节奏"、"情绪面冲击"），只要答案需要 A 股具体公司层数据（估值分位 / 季报传导进度 / 公告细节），**仍是 dual**。

例证（`rsc_gen_249`）：
> "黄金 ETF 大额申购持续，对 A 股黄金股估值修复节奏影响？"
> 表面上是"估值修复节奏"——抽象。
> 但答案需要交叉验证山东黄金/中金黄金当前估值分位 + 季报里金价传导进度。
> event 源给不出这些数据 → **dual_primary**。

例外（`rsc_gen_267`）：
> "海南自贸港加工增值免税落地，对企业实际利用率到位没有？"
> "企业"未指 A 股，答案不需查具体 A 股公司披露。
> → 倾向 **event_primary**。

## 5. 模糊时怎么定

按 spec 准则：

> 模糊时按"首个必要检索动作"判断

具体操作：
1. 把 query 拆成"哪些步骤需要什么数据"
2. 标出每一步所需数据的来源（event / disclosure）
3. 如果第一步就已经需要 disclosure → disclosure 或 dual
4. 如果第一步纯 event → event
5. 如果答案需要 event 补 disclosure 链 → dual

---

## 6. 数据生成多样性原则（避免内源性同质化）

训练数据生成时，**严格避免笛卡尔积**。

### 易踩的笛卡尔

| 反模式 | 例 | 修正 |
|---|---|---|
| 14 事件 × 6 套固定模板 | "红海最近怎么样 / 最新进展 / 意味着什么 / 为什么 / 传递什么信号 / 哪些方向"×14 事件 = 84 条几乎模板化 | 扩事件池到 30+，扩问法池到 20+，每个事件 / 问法出现次数不超过 3 |
| 18 公司 × 5 套固定模板 | "X 公司 Q3 业绩怎么样 / 分红方案力度如何 / 股权激励解锁条件" × 18 公司 = 90 条 | 扩公司池到 60+，每家公司问题数 1-2 为主，避免全部按"业绩+产能+分红+股权+其他"五件套覆盖 |

### 长尾分布建议

| 层级 | 出现次数 | 占比 |
|---|---|---|
| 高频事件 / 公司 | 4-6 条 | 5 个左右 |
| 中频 | 2-3 条 | 8-10 个 |
| 中低频 | 2 条 | 12-15 个 |
| 低频（绝大多数）| 1 条 | 40+ 个 |

目的是让模型见过数据后，不会形成"按出现次数就能猜 label"的内源性 shortcut。

### 措辞多样化

- ❌ 避免胶水句式："意味着什么"、"传递什么信号"、"利好哪些方向/板块"、"最新进展有哪些"、"最近情况如何"
- ✅ 真实 query 风格：研报式（"对 X 这轮节奏的影响传导怎么走"）、论坛式（"X 这种情况被压制了多长时间"）、雪球式（"这票还能不能拿"）、财经媒体式（"X 拐点读懂了没"）

### query 措辞独立性

同一事件/公司的多条 query 之间，措辞必须独立，不能用同一句式只换主体词。

## 7. Schema 规范（每条 JSONL 严格按此）

```json
{
  "sample_id": "rsc_gen_001",
  "query": "你的中文 query 文本",
  "intent": "event_impact_analysis",
  "event": "从 query 抽出的事件名，空则填空字符串",
  "themes": ["主题1", "主题2"],
  "target": "query 中的目标对象（公司/板块/行业）",
  "time_scope": "recent",
  "session_topic": "",
  "label": "三类标签之一",
  "label_source": "human_authored",
  "notes": "简短的判定理由，1 句话即可"
}
```

### 固定值字段

- `intent` 固定 `"event_impact_analysis"`
- `time_scope` 固定 `"recent"`（除非 query 明确不是最近事件）
- `session_topic` 固定 `""`
- `label_source` 固定 `"human_authored"`

### sample_id 命名

`rsc_gen_NNN` 前缀，3 位数字递增（001, 002, ..., 280+）。

---

## 8. 分布目标（弹性）

总数 280 左右，三类比例不强求严格均衡：

| label | 目标条数 | 实际可接受区间 |
|---|---|---|
| `event_primary` | ~93 | 80 - 130 |
| `disclosure_primary` | ~93 | 80 - 110 |
| `dual_primary` | ~94 | 60 - 110 |

真实金融 query 分布里 event 类天然多一些（政策/宏观 query 占比大），所以 event 可以接受到 130，但任何一类不应超过 130 或低于 60。

### 分布调整策略

如果某类显著超出区间：
- **优先扩池**（引入新事件 / 公司）而非删样本
- **若扩池后仍不均衡**，考虑从类别内"边缘样本"（如同时含 event 和 dual 语义的）改判
- **严禁** 因分布平衡而牺牲语义正确性

## 10. offline eval 反馈循环

训练完成、分类器替换 Stub 之后：

1. **运行 offline eval fixture**（6 条样本，含 `event_001/002`、`disclosure_001/002`、`dual_001/002`）
2. **断言失败样本回流**到冲突池
3. **重点检查**这些边界：
   - 抽象市场概念问法（情绪 / 估值修复 / 逻辑）→ 是否被误判 event
   - 提到 A 股板块但答案需要公司层数据 → 是否被误判 event
   - 跨国公司 / 政策意图主语但实际隐含 A 股 → 是否被误判 dual
4. **按本文档第 4、5 节判别**冲突样本，回灌训练数据

## 附录：典型样本示例

### event_primary（92-94 条）

- "红海局势最近怎么样了？"——纯事件状态追踪
- "OPEC+ 减产协议最近情况如何？"——纯事件追踪
- "美联储议息这周要开了，现在市场对降息节奏的预期走到哪了？"——事件+预期
- "全球降息潮背后的主要驱动是什么？"——宏观驱动解释
- "国务院最新提振消费政策传递什么信号？"——政策信号解读
- "上海自贸区本外币一体化政策落地，对跨国公司资金运作影响有多大？"——政策对非 A 股主体

### disclosure_primary（92-94 条）

- "宁德时代 Q2 毛利率是不是被碳酸锂价格压住了？"
- "比亚迪海外门店这一波扩张，单店日均销售比国内高还是低？"
- "中芯国际 N+2 节点突破以后，对应客户结构会不会出现明显切换？"
- "工业富联北美大客户 AI 服务器订单确认节奏怎么走？"
- "中航沈飞这一轮产能扩张节奏，对应下游订单储备匹配度怎么样？"

### dual_primary（~94 条）

- "美联储 9 月降息预期反复，A 股哪些方向是直接 beta？"
- "胡塞武装袭击再加码，对集运板块逻辑支撑还能持续多久？"——**即使表面"逻辑"也是 dual**（需查招商轮船/中远海控季报）
- "黄金 ETF 大额申购持续，对 A 股黄金股估值修复节奏影响？"——**即使"估值修复节奏"也是 dual**（需交叉验证山东黄金 / 中金黄金估值分位 + 季报传导）
- "创新药 BD 出海纪录被刷新，对头部 biotech 估值修复节奏？"——**dual**
- "光伏反内卷对阳光电源/迈为股份订单确定性影响多大？"——具体公司清单 dual
