# 检索策略分类器标注手册

## 任务目标

给定用户原始 `query`，判断 `collect_event_context` 应采用的起手检索策略：

- `event_primary`：优先查事件搜索源
- `disclosure_primary`：优先查披露搜索源
- `dual_primary`：两种来源都明显必要

本版采用 **query-only 标注口径**：

- 标注时只看用户原始问句
- 不依赖 `event`、`themes`、`target`、`time_scope`、`session_topic` 等派生字段
- 训练样本中也不再保留这些派生字段，避免 train / inference 特征不一致

## 主判定问题

1. 这个 query 首先是不是要先理解外部事件本身？
2. 这个 query 首先是不是要先看公司、公告或披露？
3. 这两件事是否都明显必要？

## 判定准则

### `event_primary`

满足以下任一倾向：

- 不先理解事件，就无法继续回答
- query 没有明显单公司、公告、披露中心
- 公司层或标的层只是后续可能扩展，不是第一步主问题
- 宏观/政策/地缘事件对 A 股整体、板块、风格、流动性、估值、情绪、风险偏好的广义影响，首跳先看事件与传导机制即可

重点说明：

- 只要问题还停留在“市场层面、板块层面、机制层面”的广义影响判断，即使提到了 `A 股`、`板块`、`黄金股`、`消费股` 等，也优先标 `event_primary`
- 这类问法只有在后续继续追问具体公司、订单、业绩、披露验证时，才再进入更重的公司证据链路

示例：

- `红海局势最近怎么了？`
- `美国新关税政策主要影响哪些方向？`
- `美元指数的上升会对 A 股产生哪些方面的影响？`
- `中东局势对 A 股黄金股影响怎么看？`

### `disclosure_primary`

满足以下任一倾向：

- query 本身就是公司内生事件
- 主语已经明显落在公司披露
- 不需要先大量补外部事件背景

示例：

- `宁德时代扩产公告意味着什么？`
- `某公司业绩预告是否超预期？`

### `dual_primary`

满足以下任一倾向：

- query 同时在问外部事件和 A 股公司/标的层面的落地影响
- 任意单一来源都明显不够
- 只看外部新闻或只看披露，都会丢掉关键上下文

重点说明：

- 只有“明确落到具体公司、候选标的、公司级业绩/订单/利润弹性、披露验证”的问法，才优先标 `dual_primary`
- 不能因为 query 提到了 `A 股`、`板块`、`概念股`，就机械地标成 `dual_primary`

示例：

- `红海局势升级利好哪些A股航运股？`
- `关税升级对哪些出口链公司冲击最大？`

## 边界处理

- 模糊 query：按“第一步必须做什么检索”判断
- 如果事件理解和披露检索同等必要，标 `dual_primary`
- 不使用 `unknown`、`other` 等兜底标签

## 样本字段

每条样本保留以下字段：

- `sample_id`
- `query`
- `label`
- `label_source`
- `notes`

其中：

- `query` 是唯一的判定输入
- `notes` 仅作人工说明，不作为训练特征

## 复核流程

1. 首标
2. 抽样复标
3. 争议样本进入冲突池
4. 裁决后沉淀回本手册

`label_source` 使用：

- `human_authored`
- `human_reviewed`
