FinSight Agent · 项目结构关系图

A 股事件影响分析多模块智能体系统：FastAPI 风格骨架入口 + Streamlit 工作台 + 共享 dataclass 契约；已闭环 control plane / retrieval / structured data / reporting / event_eval 五条主链。

仅供可视化参考，不修改任何源码；图谱数据生成于 2026-07-05
① 工程层 ↔ 实现层
frontend / shared / backend 三层 + 后端内部 control_plane / capabilities / infra / config / evaluation 五层；依赖方向仅 frontend→shared→backend 与 backend→shared，单向无回边。
② shared/contracts 是中枢
11 个契约 + 6 个枚举被前后端、控制面、能力层、评测模块共同消费；中心度极高，可作图谱枢纽。
③ event_eval 评测子模块
路径 backend/src/finsight_agent/evaluation/event_eval/：models + checks(6) + fixture_loader + replay + fixtures 五件套；replay 复用 WorkbenchBackendApiService 跑真实链路回放。
④ stage runners 数据驱动
6 个 run_*_stage 函数 + STAGE_RUNNERS 字典调度，由 OrchestratorService 编排，对图谱非常友好。
⑤ spec 覆盖度全闭环
11 个 spec 全部能找到实现 + 单测 + 集成测；唯一无代码落点的是 parallel-delivery-governance（纯治理规范）。
